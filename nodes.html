MyntNote Open Nodes

What is an Open Node?

Use these at your own risk!

The MyntNote software architecture allows for an easy way to use MyntNote without downloading the entire blockchain. This is called "using a remote node". Essentially, your wallet program connects to someone elses myntd (blockchain) program, a.k.a "node" a.k.a. network service

It is somewhat slower, and there is some information leak, as detailed here.

That being said, no one can steal your MyntNote because you use a remote node.

You should really run your own node

Its honestly not that hard - it can take 40 minutes to synchronize if you have a SSD and a good internet connection. Also, these remote nodes are NOT reliable. A lot of times they just won't connect, because the software really wasn't made for this.

Download your node software here: MyntNote.org and learn how to run the software here


MyntNote maintains two different collections of open nodes. One is a collection of random nodes - anyone can offer their node up for inclusion, and MyntNote will detect it and include it. These are the Random Nodes seen below.

The second type of open node are those listed as web addresses, like node.MyntNote.com. There are limited nodes in these addresses, and their status isn't checked (yet). So they may be down.

Please consider donating if you use these open nodes.

Random Nodes!

MyntNote runs a script every 30 minutes to scan the MyntNote network for open nodes. Click the button to get an IP address of a random one!

Note that these nodes use port 18089, not the standard 18081.

Windows

mynt-wallet-cli.exe --daemon-address ADDRESS.OF.HOST:18089

Unix-type (ubuntu, MacOS)

If you would like to offer your node up for inclusion in the MyntNote Network of Open Nodes, you have two options.

Option 1: you can simply add the following code to the launch of your daemon (and open your firewall), and the MyntNote script will sniff you out and add your node to the random list

--rpc-bind-ip YOUR.EXTERNAL_IP.GOES.HERE --rpc-bind-port 18089 --restricted-rpc --confirm-external-bind

Option 2: You can ask to be added to one of the address collection. There is limited space in these, so I only add high reliability nodes. You still need to launch your daemon with some special flags, and then please email me at gingeropolous@tutanota.com to ask me to add the node.

--rpc-bind-ip YOUR.EXTERNAL_IP.GOES.HERE --rpc-bind-port 18081 --restricted-rpc --confirm-external-bind
